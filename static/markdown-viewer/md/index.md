# 编程学的是什么？

让我们先回顾中国编程教育发展历史。

1984年2月16日，正值元宵佳节，邓小平在上海展览馆参观上海市微电子技术及其应用汇报展览时提出，“计算机的普及要从娃娃抓起”，青少年编程教育逐步成为公众关注的热点。2017年7月，国务院发布《新一代人工智能发展规划》，其中特别提到的“在中小学阶段设置人工智能相关课程，逐步推广编程教育”，给青少年编程教育的重要性一锤定音。提起编程教育，我们最先会想到如Scratch、Python等编程语言。是不是学会这些语言，就是学会了编程？到底学会多少语言，才算是学会了编程？想找到这些问题的答案，我们不妨从编程的发展历程中探寻。

## 0.编程是什么？
曾经在网络上看到这么一个问题
![](https://flying.zj.cn/_uploads/photos/b9Eg9cweIP5uztQr.png)
对啊，为什么输入一些单词，计算机就会做相应的事情呢？难度计算机有自己的意识吗？有人会说这是人们已经设定好的，那这个设定最初又是怎么来的呢？

我们可以认为：最原始的设计是一堆开关，**开是1，关是0**，用来控制电流，继而控制机器，可以理解为**机械式的控制**；
但是有一些很聪明的人发现**半导体硅**的高低电平可以模拟这些0和1，并且能快速精准控制，于是进化为了**电子控制**；
又有一些很聪明的人发现逐个控制太麻烦了，于是将指令用代码表示，**一条代码**通过预设好的“翻译”变为0和1让机器去执行，**汇编**就诞生了；
然后又有一些很聪明的人想到指令代码很难看懂太抽象了，能不能封装成**更自然点的话**，于是**编程语言**诞生了；
之后又有一些很聪明的人发现**磁性**可以模拟01，所以**光盘**诞生了，信息得以海量存储；
发现**电磁信号**的峰谷能模拟01，所以**无线通信**诞生了，人类做到了传说中的“遥感术”；
发现**光的不同偏振态**也可以模拟0和1，于是**光纤**通信被创造出来，信息得以光速传输；
还是这些很聪明的人觉得让机器只懂执行指令还不够，应当让它们能自己“思考”如何组织指令，于是**“算法”与“数据结构”**的概念被提炼出来，信息学编程渐渐成为一门具备完善理论体系的学科；
再后来，有人意识到单台机器的能力终究有限，为何不把千万台机器连在一起，让它们彼此对话？于是**互联网**诞生了，0和1的洪流从此跨越山海，汇聚成无形的大陆，从比尔盖茨、乔布斯开始，到国内的百度、腾讯、deepseek……创造了无数个传奇的财富神话；

历经几代人的智慧，信息学的成长史俨然就是一部0和1的演变史。当中有太多的故事成为了人们的记忆又或是早已被忘却，但无论如何，是它们造就了信息学的今天。

而未来，或许会有这样一群人：他们不再追问如何用新介质模拟0和1，而是回头审视——在开关开合之外，是否还存在另一种全然不同的逻辑，足以描绘这个宇宙更深层的真相？

## 1. 历史中的可编程发明

如果被问到“什么是编程？”，许多人的回答可能是“敲代码”。坐在电脑前，用键盘敲出一行行代码，这就是我们现在最常看到的别人编程的样子。

然而，在80年代，计算机工程师编程并不是“敲代码”，而是“写代码”——真的在纸上写出程序代码。在纸上写好的代码也不能运行，工程师们还需要把程序制作成打孔纸带，把打孔纸带输入计算机，计算机才能读取程序并运行。从1943年第一台现代电子计算机发明开始，给计算机输入打孔纸带就是当时的“编程”方式。直到电子显示器发明并大规模应用后，纯数字化的代码输入得以实现，打孔纸带才退出了“编程”的舞台。
![打孔纸带，也是早期的编程语言](https://flying.zj.cn/_uploads/photos/Imb6zaqlrPOv90kd.png)
但有趣的是，使用打孔纸带、纸卡编程的历史比现代计算机还要早得多。1805年，拿破仑为法国纺织商人、发明家约瑟夫·雅卡尔颁发巨额奖金，嘉奖雅卡尔发明的，极大提升了纺织生产效率的自动化“可设计”织布机。
![](https://flying.zj.cn/_uploads/photos/2sqjFpUfcd5M5GAA.png)
后来的各种计算机的设计，都借鉴了雅卡尔织布机的打孔卡片控制方法。十九世纪英格兰数学家查尔斯·巴贝奇，是公认的第一个机械式可编程计算机——分析机的发明者。他为了能更快的制作更准确的对数表等数学用表，设计制造了一系列“计算机”。巴贝奇就采用了打孔卡片来为分析机编写运算程序。他的助手艾达为分析机编写了完整的三角函数计算、级数相乘计算和伯努利数计算等程序。
![](https://flying.zj.cn/_uploads/photos/3d4yHDej946N3DmT.png)

## 2. 语言的进化：从“密码”到“对话”

现在，让我们从那些“会思想的机器”回到我们更熟悉的世界。你可能已经发现，无论是织布机的打孔卡片，还是计算机的纸带，它们本质上都是一种**“密码”**——人类把复杂的想法，简化成机器能懂的符号。

但总是用“密码”和机器交流，实在太累了！这就好比你想让朋友帮你买杯奶茶，却必须写下一本厚厚的操作手册：“向前走5步，左转，伸手握住门把，向下按压15度……”别说朋友了，你自己都会疯掉。

所以，一些聪明的“翻译官”出现了。

### 2.1 第一代翻译官：汇编语言——“给机器起小名”

最早的计算机程序员，用的就是由0和1组成的“机器语言”。这简直是天书！于是，人们想了个办法：给那些最常见的01组合起个简单的“小名”。

比如，用 `ADD` 代表“加”，用 `MOV` 代表“移动数据”。这种用简单单词代替01密码的语言，就是**汇编语言**。它就像把“00011101”这个密码，改名叫“开门”，好记多了！但汇编语言还是和硬件绑得很紧，不同型号的计算机，可能需要不同的“小名手册”。

### 2.2 第二代翻译官：高级语言——“说人话”的梦想

人们很快又不满足了：“为什么我们要迁就机器，用它的‘黑话’？能不能让机器来理解我们说的话？”

于是，编程界的“文艺复兴”开始了！一群程序员决定创造更接近人类自然思考方式的语言。这就好比，我们不再说“开门、拿钱、走5步”，而是直接说：“帮我去买杯奶茶！”

**故事时间：两个程序员的“厨房危机”**

让我们想象一个场景：1969年，贝尔实验室的厨房里，两位程序员——肯·汤普逊和丹尼斯·里奇，正对着咖啡机发愁。

肯：“见鬼！我们好不容易在PDP-7电脑上写好了UNIX系统，现在要把它搬到新买的PDP-11上，简直像把一本英文小说用俄语字母重写一遍！”（他们当时在用汇编语言，每种电脑的指令都不同。）

丹尼斯搅拌着咖啡，突然说：“也许……我们不该每次都重写。我们需要一种**万能翻译器**。一种语言，既能像汇编一样强大高效，直接指挥硬件，又能像写故事一样清晰流畅，让人一看就懂。”

肯眼睛亮了：“就像用标准的菜谱，无论在煤气灶还是电磁炉上，都能做出一样的菜？”

“没错！”丹尼斯在餐巾纸上画了起来，“我们需要一种语言，它自己带着‘厨房工具’（数据类型），有清晰的‘做菜步骤’（逻辑结构），但最后都能变成机器懂的‘开火、翻炒’（底层指令）。”

就这样，为了把他们心爱的UNIX系统更优雅地“移植”到新电脑上，C语言在咖啡香中诞生了。它平衡了“效率”与“可读性”，像一把瑞士军刀，既锋利又多功能。用C重写UNIX后，系统变得无比稳定和强大，最终赢得了图灵奖。C语言也成为了构建现代数字世界基石的工具。

### 2.3 第三代翻译官：面向对象与互联网时代——“乐高积木”与“世界语”

随着程序越来越复杂，“说人话”还不够，还得“像人一样组织思想”。于是，像C++、Java这样的语言带来了**面向对象**的思想。想象一下：

- **以前（面向过程）**： 造一辆车，你需要从“造一个螺丝”开始，一步步下达几千个指令。
- **现在（面向对象）**： 你有了“轮子”、“发动机”、“方向盘”这些现成的、可以重复使用的“积木块”（对象）。造车就是把这些积木按照图纸（设计）组装起来。修车时，也只需要换掉坏掉的积木块，而不是重造整辆车。

**另一个故事：一杯咖啡引发的“世界语”**

时间来到1990年代，SUN公司的一群工程师，包括詹姆斯·高斯林，正在为一个超前项目头疼：为智能有线电视盒开发软件。这些盒子芯片各式各样，每次开发都像为不同国家的人定制不同的说明书。

一天，高斯林望着办公室外茂盛的**橡树（Oak）**，又看看手里跨平台编译失败的代码，灵感突现：“为什么我们要为每个平台辛苦翻译？我们应该创造一种‘世界语’！写一次程序，就能在任何设备上运行。”

他们梦想的语言，就像咖啡豆（源代码），经过一次烘焙编译，变成咖啡粉（字节码）。任何设备，只要有台“咖啡机”（Java虚拟机），就能把它冲成可运行的咖啡。尽管智能电视项目夭折了，但历史给了他们更大的舞台——**互联网**爆发了。

当网络需要一种能安全地在不同电脑间穿梭的语言时，Java带着“一次编写，到处运行”的旗帜完美登场，迅速成为构建企业级系统和网络应用的霸主。它的名字也从“Oak”，变成了象征着“醇香与流行”的**Java**（一种咖啡）。

## 3. 内核与思维：比语言更重要的东西

读到这里，你可能会有点眼花缭乱：语言这么多，从C到Java，还有后来的Python、JavaScript……我到底该学哪个？哪个十年后不过时？

让我们做一个思想实验：假如你现在要学做饭。

- **选项A**： 只背一本《川菜菜谱》，把鱼香肉丝、宫保鸡丁的步骤倒背如流。
- **选项B**： 学习“火候”、“调味”、“刀工”这些核心原理，了解不同食材的特性。

你会选哪个？显然，选B的人，给他任何一本新菜谱，任何新厨具，他都能更快上手。甚至没有菜谱，他也能创造新菜。

编程语言，就是那一本本**菜谱**。而编程教育真正要传授的，是那些**核心的烹饪原理**。

### 3.1 编程的“内核四要素”

无论语言如何变化，所有程序都离不开这四大核心要素，它们是你与计算机对话的“语法根基”：

1.  **数据结构：信息的“收纳术”**
    计算机记性虽好，但很死板。你需要告诉它信息如何存放。就像衣柜，是把所有衣服堆成一团（数组），还是分门别类挂好（链表、树）？不同的“收纳方式”，决定了找一件衣服是花1秒钟还是1小时。学习数据结构，就是学习如何为你的数据设计最高效的“衣柜”和“仓库”。

2.  **算法：解决问题的“路线图”**
    这是编程的灵魂。假设你要从学校去机场，算法就是你的导航方案：是坐地铁（快但贵），还是公交（慢但便宜）？是换乘一次，还是直达？经典算法如排序、搜索，就像导航里的“最优路径算法”，是经过千锤百炼的智慧结晶。好的算法，能让计算机用最短的时间、最少的资源完成任务。

3.  **逻辑与控制流：程序的“决策脑”**
    程序不是僵硬的清单，它需要做判断。`如果(if)`下雨，就带伞；`否则(else)`，就不带。`重复(for/while)`搅拌面团，直到它变得光滑。这些`如果`、`否则`、`重复`，就是程序的控制流，赋予了计算机基本的“思考”和“循环”能力，让它能应对复杂多变的情况。

4.  **抽象与封装：管理的“魔法”**
    当程序变得庞大，像一座城市时，你不能事无巨细地管理每一块砖。你需要“抽象”——把复杂的细节隐藏起来，只提供简单的接口。比如，你不需要知道发动机如何工作，只需知道“踩油门车会走”。这就是封装。函数、类、模块，都是帮助我们管理复杂性的工具，让我们能像搭积木一样构建庞大系统。

### 3.2 终极目标：计算思维

掌握了这些内核，你将获得一种更宝贵的能力——**计算思维**。这不是编程的专属，而是一种普适的解决问题的方法：

- **分解**： 把大问题拆解成小问题（就像把造车分解为造轮子、造发动机）。
- **模式识别**： 找出问题中的规律（发现很多步骤其实重复）。
- **抽象**： 忽略次要细节，抓住核心模型（设计“汽车”这个抽象概念）。
- **算法设计**： 为解决问题设计清晰的步骤。

拥有计算思维的人，在面对任何复杂问题时——无论是规划一次旅行、组织一场活动，还是分析社会现象——都会下意识地运用这些方法，让思考更清晰、更高效。

## 4. 超越代码：编程赋予我们的新视角

最后，编程教育还有一个更深层的礼物：**一种理解世界的新视角**。

学会编程后，你眼中的世界会变得不一样：
- 看到交通信号灯，你会想到背后的**控制循环**。
- 使用手机APP，你会想象它背后的**数据如何流动**。
- 听说人工智能，你会理解它是**算法与数据**的结合。

编程让你从数字世界的“消费者”，变为“理解者”，甚至“创造者”。它让你明白，我们身边的智能并非魔法，而是人类逻辑与工程智慧的结晶。

从雅卡尔的织布机到今天的AI，编程的历史，就是人类不断寻找更优方法，将创意转化为现实的历史。学习编程，不仅仅是学习一门技术，更是**继承这种化想法为现实的创造力**。

所以，当你下次再看到一行代码，或使用一个智能设备时，希望你能会心一笑。因为你看到的，不再是一串神秘的字符或一个黑盒子，而是一段跨越时空的人类智慧，一场仍在继续的、激动人心的对话。
